One tiny model (a single sigmoid neuron) learns to predict who survived the Titanic, using 160 real passengers from the public Titanic dataset . We train it twice on exactly the same data: once scoring its mistakes with the quadratic loss (MSE) and once with cross-entropy. Both models start confidently wrong. Press play and watch cross-entropy recover in about 20 epochs while the quadratic model stays stuck for hundreds.

Reference notes (read after playing)

Why cross-entropy learns faster

Take one passenger who survived. The x-axis is what the model currently predicts for them; the height is how hard that mistake pushes the weights. On the far left the model is confidently wrong: cross-entropy pushes hardest exactly there, while the quadratic signal fades to almost nothing, so the MSE model stays stuck.

The maths (for the curious)

The model is a single neuron (logistic regression). For each passenger it combines four facts into one number z, then squashes it into a probability p with the sigmoid:

female?ageclassfare →weighted sum z →sigmoid →p = chance of survival
z = w1·female + w2·age + w3·class + w4·fare + b     p = 1 / (1 + e−z)

Training is plain gradient descent (learning rate 0.5, full batch): after each epoch every weight moves a small step in the direction that reduces the loss. The only difference between the two models is the loss being reduced, and therefore the push each mistake sends back:

Quadratic (MSE): L = (p − y)²  →  push on z = 2(p − y)·p·(1 − p)
The marked factor is nearly zero whenever p is near 0 or 1, including when the model is confidently wrong (exactly when it most needs correcting), so learning stalls.
Cross-entropy: L = −[y·ln p + (1 − y)·ln(1 − p)]  →  push on z = p − y
Biggest exactly when the model is most wrong. That is why cross-entropy is the standard loss for classification.

Ages, classes and (log-)fares are standardised before training, which is why one learning rate suits all four weights. Every number on screen is computed live in your browser (nothing is pre-recorded).
